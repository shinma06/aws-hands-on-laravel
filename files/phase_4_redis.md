# Phase 4: Redis組み込み

**ステータス**: 未着手

---

## 目的
OctaneのWorkerプロセス間でセッション・キャッシュを共有するためにRedisを導入する。

## 主な用途
| 用途 | 説明 |
|---|---|
| セッション | HTTPセッションをRedisに保存(Octane Worker間で共有するため必須) |
| キャッシュ | クエリキャッシュ等 |
| キュー | 非同期処理(Phase 6のSESメール送信で活用) |

## 実装方針(コスト比較)

| 方針 | コスト | 特徴 |
|---|---|---|
| ECS サイドカー(redis:alpine) | 無料 | 同一タスク内に追加 / 永続化なし / タスク再起動でデータ消える |
| ElastiCache(cache.t3.micro) | 月$15前後 | マネージド / 永続化あり / 本番相当 |

→ コスト最小ならサイドカー推奨。本番相当の学習ならElastiCache。
